Dotfiles & Shell History Logging

A public version of my configuration files are available on github. This contains Git, Bash, Zsh, and vim configurations, as well as customs scripts that are designed to feel like extra shell commands.

What is a Dotfile?

A dotfile is, by convention, a configuration with a . at the start of the file name.

Shell History Logging

I setup my shell history saving system to organize log files by device and time. When a shell is exited

BASH

# get the shortened version of the hostname
hname=`hostname -s`
# create a directory for the device if one does not exist
if [[ ! -d ~/.files/dotbash/logs/$hname ]]; then
	mkdir ~/.files/dotbash/logs/$hname
fi

# get the log file for the month
log_file=~/.files/dotbash/logs/`hostname -s`/`date +%Y%m`.log
# pipe history through awk to remove extra history index
history | awk '{ $1=""; print }' >> $log_file
# sorts by date and removes duplicates (won't double store the same command)
sort -u $log_file -o $log_file

ZSH

# get the shortened version of the hostname
hname=`hostname -s`

# create a directory for the device if one does not exist
if [[ ! -d ~/.files/dotzsh/logs/$hname ]]; then
	mkdir ~/.files/dotzsh/logs/$hname
fi

# get the log file for the month
log_file=~/.files/dotzsh/logs/`hostname -s`/`date +%Y%m`.log
# pipe history through awk to remove extra history index
history -E | awk '{ $1=""; print }' >> $log_file
# sorts by date and removes duplicates (won't double store the same command)
sort -u $log_file -o $log_file
